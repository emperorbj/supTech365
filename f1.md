FIA-F1: Design System & UI Framework Setup
Based on the PRD, PTD, and PDD documents, here are comprehensive design system recommendations aligned with your project's principles and constraints.

2. Typography
Font Stack
/* Primary: System fonts for performance & reliability */
--font-sans: -apple-system, BlinkMacSystemFont, 
             "Segoe UI", Roboto, "Helvetica Neue", 
             Arial, sans-serif;

/* Monospace: Reference numbers, IDs */
--font-mono: "SF Mono", Monaco, "Cascadia Code", 
             "Courier New", monospace;

Type Scale
// Headings
H1: 30px / 38px line-height / 700 weight (Page titles)
H2: 24px / 32px / 600 (Section headers)
H3: 20px / 28px / 600 (Subsections)
H4: 18px / 26px / 600 (Card headers)

// Body
Body Large:   16px / 24px / 400 (Primary content)
Body:         14px / 20px / 400 (Default text)
Body Small:   12px / 18px / 400 (Metadata, helpers)

// Special
Label:        14px / 20px / 500 (Form labels)
Caption:      12px / 16px / 500 (Captions, badges)
Mono:         14px / 20px / 400 (Reference numbers)

Usage Guidelines
Page Titles (H1): Workspace name (e.g., "Compliance Workspace")
Object Headers (H2): Report ID, Case reference (e.g., "Report FIA-2026-0123")
Section Headers (H3): Workflow stages, tab labels
Form Labels: Mandatory fields marked with red asterisk (*)
Reference Numbers: Always use monospace for visual distinction
Rationale:
System fonts ensure reliable rendering on FIA infrastructure (local deployment per PTD Section 11.1)
Clear hierarchy supports "users always know what object they're acting on" (PDD principle)
Readable at minimum 1366x768 resolution (NFR-U2)

7. Accessibility Guidelines
7.1 Color Contrast (WCAG 2.1 AA)
Minimum ratios:
- Normal text: 4.5:1
- Large text (18px+): 3:1
- UI components: 3:1

Testing:
- Primary Blue on White: 8.59:1 ✓
- Gray-600 on White: 7.53:1 ✓
- Error Red on White: 5.95:1 ✓

7.2 Keyboard Navigation
- All interactive elements: Tab order
- Modal focus trap: Tab cycles within modal
- Escape key: Close modals/dropdowns
- Enter/Space: Activate buttons
- Arrow keys: Navigate lists/tables

7.3 Screen Reader Support
- Semantic HTML: <button>, <nav>, <main>, <form>
- ARIA labels: aria-label, aria-describedby
- Live regions: aria-live for status updates
- Role attributes: role="alert" for errors

7.4 Form Accessibility
- Labels: Always visible, associated with inputs
- Error messages: Immediate, specific, actionable
- Required fields: Visual (*) + aria-required="true"
- Instructions: Visible before interaction


8. Responsive Behavior
Breakpoint Strategy
Mobile (<768px):
- Single column layouts
- Stacked navigation
- Full-width modals
- Simplified tables (priority columns only)

Tablet (768-1024px):
- 2-column grids where applicable
- Condensed navigation
- Adaptive modals (80% width)

Desktop (≥1024px):
- Full layouts per templates
- Side navigation option
- Multi-column grids

Component Adaptations
Tables → Cards on mobile
  - Show key fields only
  - "View Details" expands full record

Navigation → Hamburger menu on mobile
  - Slide-out drawer
  - Full hierarchy visible

Forms → Single column on mobile
  - Full-width inputs
  - Stacked buttons


9. Animation & Transitions
Timing Functions
Default: cubic-bezier(0.4, 0.0, 0.2, 1) -- "ease-out"
Enter:   cubic-bezier(0.0, 0.0, 0.2, 1) -- "ease-in"
Exit:    cubic-bezier(0.4, 0.0, 1, 1)   -- "ease-out"

Durations
Fast:   150ms (hover, focus states)
Normal: 250ms (default transitions)
Slow:   350ms (complex animations)

Animation Guidelines
✓ Use for:
  - State changes (loading → success)
  - Page transitions
  - Modal open/close
  - Hover feedback

✗ Avoid:
  - Gratuitous effects
  - Blocking animations
  - Performance-heavy sequences

Rationale: Align with PDD principle that system must feel responsive; users shouldn't wait for animations. All transitions should be faster than the 3-second dashboard load target (NFR-P2).
